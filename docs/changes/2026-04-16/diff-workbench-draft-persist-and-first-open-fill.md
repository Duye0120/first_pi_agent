# Diff 工作区保留草稿并补齐首次打开占位

**时间**: 2026-04-16 15:42

## 改了什么

1. 给 Diff 工作区增加了挂载外草稿状态，保留提交标题、描述、布局、视图、来源、活动文件和分栏尺寸。
2. 关闭右侧工作区后，重新打开会继续显示刚才生成的提交内容。
3. 右侧 dock 打开过渡阶段提前渲染占位宽度，首次打开时直接填满新增区域。
4. Diff 内容区外边距从 `px-4 py-4` 收紧到 `px-3 py-3`，让边缘更贴合容器。

## 为什么改

- 用户希望关闭再打开时，右侧已经生成的内容继续保留。
- 用户第一次打开时看到右侧容器四边还有一圈细小空隙，体感上像没有完全撑满。

## 涉及文件

- `src/renderer/src/components/assistant-ui/diff-panel.tsx`
- `src/renderer/src/App.tsx`

## 结果

- 生成过的提交内容会保留在 Diff 工作区里。
- 首次打开右侧工作区时，新增区域会立即被内容占满。
