# UI 间距与对齐优化

**时间**：2026-04-15 10:00:00

## 更改内容

1. 修复了侧边栏（`sidebar.tsx`）悬浮按钮大小不一致的问题。
   - 原先的设置按钮使用的是 `h-8 px-2`（高度 32px，较扁），而“新线程”和普通线程列表项高度接近 40px（使用了 `py-2.5 px-3` 等不同的 padding 组合）。
   - 现将其结构统一抽离为 `py-2 px-3 gap-2` 的响应级列表高度格式，使左侧所有导航、线程、底部操作拥有绝对相等的悬停高亮覆盖面积与图标边距对齐。

2. 修复了主对话区域（`thread.tsx`）聊天框与聊天气泡整体不对齐的问题。
   - 原本内容 Viewport 强制向右移位 32px （`pl-8` 加上 `mr-3` 等设定），而 Composer 盒子本身只退让了 20px（`px-5`），导致视觉上输入框比聊天记录更宽、向左凸出。
   - 已将整体容器拉平，统一采用 `px-6` 逻辑（两边保持 24px 网格偏移），让 Composer 与顶部所有元素的边缘结构自动在一条垂直线上。

## 为什么进行本次更新
- 用户反馈聊天窗口和底侧侧边栏设置悬浮图标不够规整，且存在尺寸拉扯。
- “不能一修错就随手去跑 npm run build 或 check”，这是用户在提示词及根目录 `AGENTS.md` 里反复重申的心智要求，以此保证迭代环境不在不必要指令中卡死。

## 后续约束
- 若继续向 Composer 或者侧边栏填入新组件，切忌硬编码定宽高度（如单拿一个 `h-8` 卡死布局），优先通过 `py` 或 `px` 进行 Flex Box 的伸缩适应。
- 没有要求编译发布时，坚决不进行 `pnpm build` 和 `pnpm check`。
