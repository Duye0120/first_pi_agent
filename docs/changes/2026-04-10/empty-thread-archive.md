# 空线程允许保留 + 最后一个线程可归档

**时间**：2026-04-10 15:20

## 改了什么

### 允许真正的空线程列表
- **原因**：当前逻辑会在“最后一个线程被归档 / 删除”后立刻自动创建一个新线程，用户永远到不了“空线程”状态，所以体感上像“最后一个不给归档”。
- **改法**：
  - `bootApp()` 在没有任何活跃线程时不再强行 `create()`
  - 归档最后一个活跃线程后不再自动补新线程
  - 删除最后一个活跃线程后也不再自动补新线程

### 首页空态补成可操作状态
- **原因**：允许空线程后，首页不能只剩一句“当前没有可用线程”或者看起来像空白，否则会像是界面坏了。
- **改法**：
  - 主内容区空态改成明确的“当前没有活跃线程”
  - 提供“新建线程”按钮
  - 如果存在已归档线程，再补一个“查看已归档”入口，直接跳到设置里的归档区

### 归档 / 恢复时清掉旧缓存
- **原因**：线程被归档或恢复后，如果 renderer 继续复用旧 session cache，后续再打开这个线程时会拿到过期的 `archived` 状态。
- **改法**：
  - 在归档 / 恢复链路里先移除对应 session cache
  - 下次打开时重新从主进程加载，避免旧状态污染

## 改到哪些文件

| 文件 | 改动 |
|------|------|
| `src\renderer\src\App.tsx` | 去掉最后一个线程归档/删除后的自动补线程逻辑；补空线程首页空态；清理 active session 存储键；归档/恢复时失效旧 cache |
