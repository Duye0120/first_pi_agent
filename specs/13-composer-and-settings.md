# 13 — Composer 增强与设置

> 状态：`in-review`
> 依赖：03-agent-core、10-steps-visualization

## 13.1 设计目标

Composer 是用户与 Agent 对话的入口。当前只有基础的文本输入 + 附件，需要增强为一个功能完整的输入工作台。Settings 页面则让用户管理 API Key、模型选择、workspace 等配置。

## 13.2 Composer 布局

```
┌─ Composer ──────────────────────────────────────────────┐

│  ┌─ 附件栏（有附件时才显示）──────────────────────────┐  │
```

## 13.3 输入区

### 多行文本输入

- 默认单行高度，输入内容增多时自动增长（最高 200px，之后滚动）
- `Enter` 发送消息
- `Shift + Enter` 换行
- `Cmd/Ctrl + Enter` 强制发送（即使正在输入中文 IME）
- 支持粘贴图片（自动添加为附件）
- 支持拖拽文件到输入区（添加为附件）

### 占位文本

根据 Agent 状态动态变化：

| 状态 | 占位文本 |
|------|---------|
| 空闲 | `给 Pi 发消息…` |
| Agent 正在执行 | `Agent 正在执行中，发送消息将在完成后处理…` |
| 无 API Key | `请先在设置中配置 API Key` |

### 草稿持久化

用户输入的内容实时保存到会话的 `draft` 字段（已有机制），切换会话时恢复。附件列表同理。

## 13.4 工具栏

### 附件按钮

点击打开文件选择器（已有 `desktopApi.files.pick`），选中的文件显示在附件栏。

附件栏中每个附件显示：
- 文件图标（根据类型：代码文件、图片、文本等）
- 文件名（过长截断，tooltip 显示完整路径）
- 删除按钮 `×`
- 图片类型的附件显示缩略图预览

### 模型选择器

下拉菜单，显示当前选用的模型，点击切换：

```
┌─ 模型 ────────────────────────┐
│  ◉ Claude Sonnet 4           │  ← 当前选中
│  ○ Claude Opus 4             │
│  ○ GPT-4o                    │
│  ○ DeepSeek V3               │
│  ──────────────────────────── │
│  ○ Ollama: llama3            │  ← 本地模型（如果配了 Ollama）
│  ──────────────────────────── │
│  ⚙ 管理模型配置…              │  → 跳转 Settings
└───────────────────────────────┘
```

- 只显示用户已配置 API Key 的 provider 的模型
- 切换模型实时生效（下一次 Agent 调用用新模型）
- 当前模型在工具栏显示为紧凑标签：`Claude Sonnet 4 ▾`

### 思考级别选择器

控制 Agent 的 thinking level（03-agent-core 3.4 节定义）：

```
┌─ 思考级别 ──────────────┐
│  ○ 关闭                 │  off — 不输出思考过程
│  ◉ 简洁                 │  low — 简短思考
│  ○ 标准                 │  medium — 默认
│  ○ 深度                 │  high — 详细推理
│  ○ 最大                 │  xhigh — 最大推理深度
└─────────────────────────┘
```

工具栏显示为图标 + 当前级别：`💭 简洁 ▾`

### 发送按钮

- 有内容时高亮可点击，无内容时灰色禁用
- Agent 正在执行时，发送按钮变为 **停止按钮**（红色方块图标）
- 点击停止 → 触发 AbortSignal 取消当前执行

## 13.5 Agent 执行中的 Composer 状态

当 Agent 正在执行时，Composer 行为变化：

| 元素 | 变化 |
|------|------|
| 输入区 | 仍可输入（预排下一条消息），但不会立即发送 |
| 发送按钮 | 变为停止按钮 |
| 工具栏 | 模型和思考级别选择器禁用（执行中不能切换） |
| 附件 | 仍可添加/删除 |

用户在 Agent 执行中输入的消息，Agent 完成后自动发送（排队机制）。或者用户可以取消排队的消息。

## 13.7 Settings 数据存储

Settings 分两类存储：

| 数据类型 | 存储方式 | 原因 |
|---------|---------|------|
| API Keys | `credentials.json`（单独文件，0600 权限） | 敏感数据，独立保护 |
| 模型/外观/终端等配置 | `settings.json` | 普通配置 |
| UI 状态（面板开关、抽屉高度等） | `desktop-shell-state.json`（已有） | 窗口状态 |

所有文件存储在 `${app.getPath('userData')}/` 目录下。

```typescript
// settings.json 结构
{
  "defaultModel": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-20250514"
  },
  "thinkingLevel": "low",
  "theme": "light",            // "light" | "dark" | "custom"
  "customTheme": { ... },      // 自定义主题的 CSS 变量映射
  "terminal": {
    "shell": "default",
    "fontSize": 13,
    "fontFamily": "JetBrains Mono",
    "scrollback": 5000
  },
  "ui": {
    "fontSize": 14,
    "codeFontSize": 12,
    "codeFontFamily": "JetBrains Mono"
  },
  "workspace": "/Users/alice/projects/my-app"
}
```

