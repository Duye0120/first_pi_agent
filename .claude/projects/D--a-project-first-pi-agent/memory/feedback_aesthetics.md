description: User believes ugly things are unusable — visual quality is a hard requirement, not nice-to-have
type: feedback
---

不好看的东西一定不好用。

**Why:** 用户对 UI 审美有高标准，视觉品质是功能可用性的前提条件，不是锦上添花。

**How to apply:** 所有前端相关的 spec 和实现，都要把视觉设计当作一等公民。选型时优先考虑美观度，不要为了"简单"而妥协视觉体验。
